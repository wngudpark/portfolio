## 프로젝트 개요

DB금융경제공모전 운영을 위한 사용자 포털입니다. foundation(재단 포털)과 공통 데이터베이스를 사용하며, 패키지는 공모전 고유 기능과 회원·장학·커뮤니티 등 공통 기능으로 분리되어 있습니다. Spring MVC와 MyBatis, JSP 서버사이드 렌더링을 기반으로 하고, 일부 상호작용은 jQuery Ajax로 처리했습니다.

## 담당 업무

- **공모전 팀 신청 플로우**: 팀장·팀원 역할을 분리하고, 팀원 등록 시 본인 확인 및 재학 여부 검증 후 신청 데이터를 트랜잭션으로 저장했습니다. 팀원 탈퇴 시에는 인증 정보로 본인 여부를 서버에서 재확인해 IDOR을 방어했습니다.
- **출품작 제출**: 파일 업로드에 확장자 화이트리스트, 매직 바이트 검증, 경로 조작 방지 처리를 적용했습니다. 신청 상태(신청중/접수완료)별로 페이지 플로우를 분기했습니다.
- **공모전 정보 페이지**: 공모요강, 공지사항(고정 공지 포함), 히스토리, 우수학술동아리, 글로벌 금융탐방, 역대 수상작 등 정보 제공 페이지를 구현했습니다.
- **증명서 PDF 자동 생성**: JSP 템플릿을 HTML을 거쳐 PDF로 변환(수혜 증명서·수상 확인서·참여 확인서 3종)했습니다. 영문판은 PDF 대신 안내 메일을 비동기로 발송하도록 처리했습니다.
- **캐시 적용**: 전역 메뉴와 공통 코드를 캐싱 처리하고, 관리자가 수동으로 캐시를 갱신할 수 있는 기능을 제공했습니다.
- **비밀번호 이중 검증**: 비밀번호 형식에 따라 신규 방식과 레거시 방식을 분기 검증하고, 만료(90일) 시 즉시 변경 또는 유예를 선택할 수 있는 플로우를 구현했습니다.
- **NICE 본인인증**: 토큰 발급과 암호화 요청·복호화 로직을 구현하고, 인증 관련 파라미터를 보안 필터에서 예외 처리했습니다.
- **인프라 구성**: foundation과 동일하게 DB Inc.에서 제공받은 Windows 서버 환경에서 WEB, WAS, DB 서버 구성을 직접 담당했습니다.

## 개선 과정

- **조회 속도 개선**: 페이지 진입마다 전역 메뉴를 DB에서 조회하던 구조에 인메모리 캐시를 도입하고, 커넥션 풀 설정을 명시적으로 추가해 성능을 개선했습니다.
- **보안 조치**: 개인정보 업로드 경로에 접근 차단을 적용하고, 잔류 파일을 정리했습니다. 또한 Ajax 미인증 요청에 대한 응답 방식을 개선해 로그인 화면이 팝업에 잘못 주입되던 오동작을 해결하고, 쿠키 보안 옵션을 직접 설정하는 방식으로 전환했습니다.
- **인증 서비스 분리**: 컨트롤러에 분산되어 있던 NICE 복호화·토큰 생성 로직을 별도 유틸/서비스로 추출하고, 업로드 보안 검증 로직을 한 곳에 집중시켜 모든 업로드 지점에 일관되게 적용했습니다.
- **더보기 후 뒤로가기 오류 수정**: 목록을 추가로 불러온 뒤 상세 페이지에서 뒤로가기 시 처음 일부만 표시되던 버그를, 이전 조회 위치를 저장해두었다가 목록 진입 시 그 위치까지 한 번에 재요청하는 방식으로 해결했습니다.
- **증명서 알림 메일 추가**: 신청 접수 확인이 어렵다는 운영 피드백을 반영해, PDF 생성 후 관리자에게 비동기로 알림 메일을 발송하도록 개선했습니다.
