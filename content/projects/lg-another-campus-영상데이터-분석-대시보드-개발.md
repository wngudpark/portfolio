## 프로젝트 개요

대학 캠퍼스 내에 운영되는 오프라인 공간(서울대·유니스트)의 방문자 동선 데이터를 수집·분석하기 위한 시스템입니다. 외부 API에서 제공되는 방문 동선 원천 데이터를 매일 배치로 수집해 시간대·공간별 지표로 가공한 뒤 적재하고, 관리자는 대학별로 분리된 대시보드에서 방문 인원, 방문 횟수, 평균 체류 시간 등을 조회·다운로드할 수 있습니다. 데이터 수집 배치와 관리자 대시보드가 하나의 데이터베이스를 공유하는 구조로 설계했습니다.

## 담당 업무

- **API 데이터 수집 배치 파이프라인**: 매일 전일자 데이터를 대학별로 순회하며 외부 API를 호출하고, 응답을 검증(HTTP 상태 코드, 빈 값, JSON 파싱)한 뒤 대학별 처리 로직으로 위임하며, 수집 성공/실패 이력을 별도로 기록하도록 구현했습니다.
- **원천 데이터 파싱·적재**: API 응답에서 날짜와 시작·종료 시간을 분리해 방문 동선을 방문자 단위로 파싱하고, 원본 데이터·첫 방문 데이터·동선 데이터로 나누어 저장했습니다. 저장 로직은 Prepared Statement로 처리해 안전성을 확보했습니다.
- **시간대별 통계 집계 생성**: 공간별 방문 인원, 방문 횟수, 평균 체류 시간, 첫 방문 인원, 첫 방문 평균 체류 시간을 시간대 단위로 집계해 별도의 집계 데이터로 적재하고, 대시보드는 이 집계된 데이터만 조회하도록 구성해 조회 성능을 확보했습니다.
- **수동 스크립트**: 단일 날짜 또는 날짜 범위를 지정해 과거 데이터를 재수집·재적재할 수 있는 별도 스크립트를 구성했습니다.
- **JWT 인증 및 대학 단위 접근 제어**: 페이지 접근 시 URL에 포함된 대학 코드와 인증 토큰의 소속 대학 정보를 대조해 권한을 검증하도록 구현했습니다.
- **통계 대시보드 및 다운로드**: 전체 방문, 첫 방문, 공간 상세, 포토존 지표를 대학별로 출력하고, 기간·시간대 필터와 비동기 조회, 엑셀 다운로드 기능을 구현했습니다.

## 주요 해결 과제

- **중복 방문자 데이터 처리**: 동일 방문자가 여러 번 집계되는 문제를 발견해, 방문자 단위로 중복 여부를 판정하는 로직을 추가했습니다. 중복으로 판정된 건은 통계 집계에서 제외하되, 실제 동선 기록은 별도 표시를 남겨 데이터 추적이 가능하도록 처리했습니다.
- **재방문 판정 로직 개선**: 동일 방문자의 연속된 동선에서 이전 퇴실 시점과 이후 재입장 시점 사이에 일정 시간 이상 간격이 있을 때만 별도 방문으로 집계하도록 수정해, 짧은 이동으로 인한 과다 집계를 방지했습니다. 포토존은 전체 체류·방문 집계에서 제외해 지표 왜곡을 보정했습니다.
- **집계값 예외 처리**: 방문자가 없는 시간대에서 발생하는 0으로 나누기 문제를 데이터 수집 단계와 조회 단계 양쪽에서 방어 로직을 적용해 해결했습니다.
- **집계 로직 리팩토링**: 대학·조회유형별로 중복되던 통계 쿼리와 출력 코드를, 매핑 기반으로 필요한 컬럼을 동적으로 조립하는 공통 함수로 통합해 유지보수성을 높였습니다.
- **배치 오류 격리**: 대학 단위 처리 구간을 예외 처리로 감싸, 특정 대학의 데이터 수집 실패가 다른 대학의 수집까지 중단시키지 않도록 구성했습니다.
