## 프로젝트 개요

DB금융그룹 재단이 운영하는 두 공개 웹사이트(foundation, dbfec)에 대한 통합 관리자 시스템입니다. 다수의 MyBatis Mapper와 엑셀 템플릿, Controller로 구성되며, 두 프론트엔드 사이트는 어드민의 알림 API를 호출해 회원 알림 이력을 기록하는 단방향 구조입니다. 어드민은 세션 기반으로 두 사이트를 전환해 각 화면 콘텐츠를 관리합니다.

## 담당 업무

- **회원 관리**: 일반·동문·장학생·블랙리스트·탈퇴 회원을 유형별로 관리하고, 개인정보처리방침·이용약관 버전을 관리했습니다.
- **장학 프로그램 관리**: 드림리더·드림마스터·재정·계속장학·지원 등 6개 유형의 지원자→1차합격→최종합격 심사 프로세스와 수혜자·이력을 관리했습니다. 유형별 엑셀 출력은 전략 패턴으로 분기 처리했습니다.
- **공모전 관리**: 참가 신청과 지원자 심사, 수상자 선정 및 관련 데이터를 관리했습니다.
- **엑셀 대량 처리**: 공통 추상 클래스와 다수의 구현체, 엑셀 템플릿을 활용해 도메인별 다운로드와, 심사 결과를 일괄 반영하는 엑셀 업로드 기능을 구현했습니다.
- **PDF 생성**: JSP 템플릿을 HTML을 거쳐 PDF로 변환하고, 스레드 풀 기반 병렬 처리로 수상증서·장학신청서를 대량 생성했습니다.
- **알림 배치**: 매일 새벽 지정된 시각에 실행되는 배치로, 장학금 마감 D-7/D-3/D-1 시점에 임시저장 상태인 회원에게 자동 알림을 발송했습니다.
- **보안**: IP 화이트리스트 필터, 로그인 5회 실패 시 계정 잠금, 90일 미사용 계정 비활성화, 세션 중복 방지, 90일 주기 비밀번호 교체 강제 기능을 구현했습니다.
- **시스템 관리**: 관리자 계정·메뉴 권한·IP 허용목록·공통코드 관리와 통계 대시보드를 구현했습니다.
- **환경설정 검증**: 필수 환경변수가 설정되지 않으면 기동 시점에 즉시 오류를 발생시켜 설정 누락을 조기에 감지하도록 구현했습니다.
- **인프라 구성**: DB Inc.에서 제공받은 Windows 서버에 WEB, WAS, DB 서버를 직접 구성하여 세 시스템(foundation, dbfec, admin)이 동일한 인프라 위에서 운영되도록 했습니다.

## 개선 과정

- **IP 스푸핑 취약점 수정**: 여러 요청 헤더를 신뢰하던 IP 필터를, 실제 접속 IP만 사용하도록 수정해 허용목록 우회를 차단했습니다.
- **경로 조작 방지**: 파일 삭제 기능의 미검증 경로 파라미터를 정규화한 뒤 허용된 경로 하위인지 검증하도록 수정하고, 비정상 요청은 오류로 응답하도록 처리했습니다.
- **하드코딩 비밀번호 제거**: 소스 코드에 평문으로 하드코딩되어 있던 관리자 초기 비밀번호를 환경변수로 이관했습니다.
- **비밀번호 검증 로직 복원**: 개발 편의를 위해 주석 처리된 채 배포되어 있던 현재 비밀번호 검증 로직을 복원해, 세션 탈취 시 비밀번호가 무단으로 변경될 수 있는 위험을 제거했습니다.
- **알림 배치 로직 개선**: 유형·학기 구분이 불가능했던 배치 로직을 유형과 학기를 함께 처리하도록 수정하고, D-7/D-3/D-1 배치를 분리 구현했습니다.
